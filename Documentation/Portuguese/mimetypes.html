MIME TYPES

Você pode modificar os MIME types usados pelo MyServer, alterando o arquivo "MIMEtypes.xml" com um editor de texto, como o notepad no Windows, ou o Emacs no Linux. O que segue é um exemplo de um arquivo .xml:

<MIMETYPES>
-
    <MIMETYPE>
-
    <MIMETYPE>
<EXT>htm</EXT>
<MIME>text/html</MIME>
<CMD>SEND</CMD>
<MANAGER>NONE</MANAGER>
</MIMETYPE>


Cada linha descreve um MIME type diferente.

<EXT> é o tipo do arquivo

<MIME> é o Mime type.

<CMD> é o modo de enviar o arquivo ao cliente

<MANAGER> descreve o comando <CMD>.

Os seguintes valores para enviar o arquivo ao cliente são válidos na seção <MANAGER>:

  1. SEND - Envia o arquivo estaticamente como ele é. Esse comando é usado para extensões de arquivo tais como .html e .jpg, que não precisam de intervenção externa do servidor web.
  2. RUNCGI - Executa o script CGI indicado no caminho. É usado para extensões de arquivo tais como .php e .pl, necessários de serem processados antes de enviar qualquer dado para o cliente.
  3. RUNFASTCGI - Roda o script Fastcgi. Esse tipo precisa de um parser de Fastcgi (veja a página do PHP). É também usado para extensões de arquivo como .php e .pl
  4. EXECUTEFASTCGI - Isso também roda o script Fastcgi, mas diferentemente do RUNFASTCGI acima, não precisa de um servidor para rodar o script, ao invés, ele age como um auto-servidor.
  5. RUNMSCGI- Roda o script MSCGI. Essa library só está disponível para o myServer. É usada para construir CGIs rápidos e eficientes.
  6. EXECUTE -Trata o arquivo como um executável e envia a saída gerada para o cliente. 
  7. SENDLINK -Envia o arquivo contido em um link.
  8. EXECUTEWINCGI - Envia o arquivo como um WinCGI
  9. RUNISAPI - Roda o script ISAPI.
  10. EXECUTEISAPI - Roda o script ISAPI mas não precisa de um servidor para executar. Ao invés disso, ele age como um auto-servidor.

A maioria das aplicações já estão listadas no arquivo .xml. Se você gostaria de adicionar um MIME type, coloque no fim do arquivo e uso o código como um exemplo.
