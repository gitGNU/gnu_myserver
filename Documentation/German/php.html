BENUTZUNG VON PHP MIT MYSERVER



PHP (Hypertext Preprocessor) ist weltweit die meistgenutzte serverseitige Skriptsprache. Eine serverseitige Sprache wird benutzt, um Webseiten mit dynamischen Inhalten zu erstellen. Man kann PHP mit MyServer benutzen, um eine standard executable CGI oder eine FastCGI executable zu erstellen. Während der Benutzung von PHP als CGI executable, erstellt MyServer einen neuen Prozess für jede angeforderte Seite. Wenn PHP im FastCGI Modus benutzt wird, braucht MyServer einen neuen Prozess nur für jede Anfrage, deshalb hat der Server eine bessere Performance. Mehr Informationen gibt es auf der FastCGI Webseite

Um PHP verwenden zu können, müssen sie zuerst den Installer von: www.php.net herunterladen.

In den aktuellen PHP Versionen müssen sie auch die php.ini Datei modifizieren und die cgi.force_redirect Option auf null setzen. Finden Sie diese Datei in der PHP Konfigurationsdatei:

;cgi.force_redirect= 0

und entfernen Sie das erste Semikolon.
Die Zeile sieht danach so aus:

cgi.force_redirect = 0

Remote FastCGI Servers

MyServer kann remote FastCGI server benutzen. Die folgenden MIME Typen benutzen eine remote connection:

<MIMETYPE>
<EXT>php</EXT>
<MIME>text/html</MIME>
<CMD>RUNFASTCGI</CMD>
;


<MANAGER>@host:port</MANAGER>
</MIMETYPE>


Die Definition für einen remote Manager muss mit einem @ beginnen.

Wenn Sie die MIME Typen mit der Control center Applikation konfigurieren, können Sie einen remote server durch das Hinzufügen einer Zeile in der Form

@HOST:PORT

in dem FastCGI Verzeichniseingabefeld tun.
