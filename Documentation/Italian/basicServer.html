CONFIGURAZIONE BASE DEL SERVER

Nel file myserver.xml si possono configurare le pagine di default per il web server.

Le linee sottostanti già esistono nel tuo file di configurazione di default:

    <DEFAULT_FILENAME0>default.html</DEFAULT_FILENAME0>
    <DEFAULT_FILENAME1>default.php</DEFAULT_FILENAME1>
    <DEFAULT_FILENAME2>default.php3</DEFAULT_FILENAME2>
    <DEFAULT_FILENAME3>index.html</DEFAULT_FILENAME3>
    <DEFAULT_FILENAME4>index.php</DEFAULT_FILENAME4>
    <DEFAULT_FILENAME5>index.php3</DEFAULT_FILENAME5>
    <DEFAULT_FILENAME6>default.htm</DEFAULT_FILENAME6>


Avendo un virtual host Virtual Host http://mypc.net, la pagina che apparirà quando lo si accede sarà la prima pagina trovata avente quel nome. Tu puoi aggiungere altre linee o modificare quelle esistenti. Lo stesso comportamento lo si ha per ogni directory browsing. Se nessuna delle pagine è presente allora il server manda il contenuto della directory al client.


Nella sezione Errori viene spiegato come personalizzare le pagine di errore. Inoltre si possono disabilitare le pagine di errore cambiando le seguente linea:
<USE_ERRORS_FILES>YES</USE_ERRORS_FILES>
Mettendo il valore "NO":
<USE_ERRORS_FILES>NO</USE_ERRORS_FILES>
il server invia un messaggio solo testo.
si può cambiare il layout della pagina di browsing (usata quando nno ci sono files di default), ulteriori informazioni sono nella sezione Directory Browsing . La seguente linea, in myserver.xml, punta al file CSS usato per la configurazione del layout:
<BROWSEFOLDER_CSS>system/folderstyle.css</BROWSEFOLDER_CSS>

Gli script CGI possono usare l'e-mail dell'amministratore del server, quindi è importante settare correttamente questo valore nella riga sottostante in myserver.xml:
<SERVER_ADMIN>administrator@localhost</SERVER_ADMIN>
