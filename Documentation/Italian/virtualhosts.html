VIRTUAL HOSTS

In teoria, i virtual hosts permettono agli utenti di ospitare infiniti siti sul proprio PC. Per esempio http://mypc.net è differisce da http://sub.mypc.net. Per definire virtual hosts con MyServer apri il file virtualhosts.xml in un editor di testo(come notepad per windows ed Emacs per linux). Un tipico virtual hosts somiglia a:

<VHOSTS>
-
    <VHOST>
<NAME>Every connection</NAME>
<PORT>80</PORT>
<PROTOCOL>HTTP</PROTOCOL>
<DOCROOT>web</DOCROOT>
<SYSFOLDER>system</SYSFOLDER>
<ACCESSESLOG>logs/myServer.log</ACCESSESLOG>
<WARNINGLOG>logs/myServer.err</WARNINGLOG>
</VHOST>
</VHOSTS>


<NAME> dà all'amministratore un idea di cosa il virtual host è come internal Ethernet, external internet o every connection. 

<PORT> la porta TCP usata per il listening
<PROTOCOL> protocollo usato dal virtual host,
<DOCROOT> la cartella dove sono i files.
<SYSFOLDER> la cartella con i files di sistema.
<ACCESSLOG> il file di log per gli accessi.
Questa riga può apparire nella forma: <ACCESSESLOG type="combined">logs/myServerHTTP.log Con questa forma vengono aggiunti due ulteriori campi nel file di log per ogni record, HTTP Referer e HTTP User Agent. <WARNINGLOG> il file di log per gli errori e i warnings.

Con MyServer è possibile attivare l'HTTP TRACE per ogni virtual host che usa L'HTTP. Per attivarlo bisogna modificare il file security nella cartella radice dell'host (non una sottocartella). Se il file non esiste, bisogna crearne uno nuovo, seguendo le indicazioni del file Security e aggiungendo i parametri

<HTTP TRACE="ON" />

Una volta fatto questo l'HTTP TRACE è attivo. Di default è disattivato per evitare problemi di sicurezza.
