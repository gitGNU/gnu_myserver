BASIC SERVER CONFIGURATION

In the myserver.xml file you can set the default page for your websites hosted in MyServer.

The lines below already exist in your configuration file:

<DEFAULT_FILENAME0>default.html</DEFAULT_FILENAME0>
    <DEFAULT_FILENAME1>default.php</DEFAULT_FILENAME1>
    <DEFAULT_FILENAME2>default.php3</DEFAULT_FILENAME2>
    <DEFAULT_FILENAME3>index.html</DEFAULT_FILENAME3>
    <DEFAULT_FILENAME4>index.php</DEFAULT_FILENAME4>
    <DEFAULT_FILENAME5>index.php3</DEFAULT_FILENAME5>
    <DEFAULT_FILENAME6>default.htm</DEFAULT_FILENAME6>


So, if you have a Virtual Host http://mypc.net, the page that will appear when accessing that address on the browser, will be one of the files above. You can add more lines or change any of them, if you need a different filename. The same mechanism is used for every directory and not only with the root one.



In Errors section you learned how to create and use your own error pages. But you can also disable the custom error pages, by changing the line below:
<USE_ERRORS_FILES>YES</USE_ERRORS_FILES>
So, if you set the value to NO, like in the line below
<USE_ERRORS_FILES>NO</USE_ERRORS_FILES>
you will have TEXT messages instead of HTML pages for the errors.

You can change the layout of the directory browsing page (used when you don't have a default page, as described in the Browse directory index section). The line below, in myserver.xml, points to the CSS file used to configure the layout of that page:
<BROWSEFOLDER_CSS>system/folderstyle.css</BROWSEFOLDER_CSS>

CGI Scripts can get the Server Administrator E-mail and show them to the user(through the SERVER_ADMIN environment variable). So, it can be important you define this address properly, by setting the value in the line below, in myserver.xml:
<SERVER_ADMIN>administrator@localhost</SERVER_ADMIN>
