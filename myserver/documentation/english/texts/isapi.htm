ISAPI


ISAPI (Internet Server Application Program Interface) est un protocole utilisé par les ordinateurs basés sur Windows pour lancer des applications dynamiques en chargeant un module externe dans l'espace d'adressage du processus du serveur web. Afin d'utiliser ISAPI, vous devez créer un DLL (Dynamic Link Library) qui permet l'export des symboles appelés par le serveur web.

Attention: Si une application ISAPI plante, cela pourrait causer le plantage de MyServer.

Si vous désirez utiliser le module ISAPI dans un module auto-contenu, vous devrez alors enregistrer l'extension, par exemple .isapi, à utiliser comme module ISAPI, et ajouter la commande EXECUTEISAPI dans le type MIME.

Si vous désirez utiliser le module ISAPI pour gérer tout les fichiers d'une extension donnée, vous devrez utiliser la commande RUNISAPI; De cette façon, tout les fichiers de cette extension seront gérés par le MANAGER spécifié.

Repportez-vous à la documentation des types MIME pour avoir plus d'informations sur la façon dont sont associés les types MIME dans MyServer.


Pour plus d'informations, vous pouvez lire l'introduction ISAPI de Microsoft.

Il est fortement recommandé d'utiliser ce protocole de communication si le module ISAPI est stable et vient d'une source de confiance, puisque le module est directement chargé dans l'espace d'adressage de MyServer, car il est plus rapide et sans surcouche pouvant amener des erreurs externes d'execution.
