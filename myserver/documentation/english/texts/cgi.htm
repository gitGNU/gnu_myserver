CGI

CGI (Common Gateway Interface) est la méthode standard par laquelle un serveur web récupère les options de l'utilisateur et retourne des informations à l'utilisateur. CGI dépend de scripts souvent écrits dans des langages tels PHP ou Perl pour effectuer des opérations demandées par l'utilisateur. Afin d'accéder aux scripts CGI, le nom de l'application est donnée dans le code HTML.

Par exemple, si vous créez un formulaire, l'URL du script est indiquée par les champs de la balise FORM:

<FORM METHOD=POST ACTION=www.myfiles.com/cgi-bin/myprog.pl>

Note: Tout langage de script affichant des données peut être utilisé en fait - vous pouvez utiliser des scripts en Script Shell, Python, etc... Vous avez juste besoin de configurer correctement le type MIME, et votre langage de script préféré fonctionnera avec MyServer.

Cependant, les scripts CGI peuvent être placés n'importe où du moment que leur emplacement est indiqué dans le code HTML et qu'ils sont configurés pour être utilisés via CGI.

Les scripts CGI créent un nouveau processus pour chaque requête. Cela peut causer des problèmes de performance mais assure une meilleure sécurité puisque le processus principal de MyServer est à l'abri d'un crash du processus CGI.
