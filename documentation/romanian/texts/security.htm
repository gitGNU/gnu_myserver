CONTROLUL ACCESULUI


Pentru a controla accesul la un fişier directorul care conţine acest fişier trebuie adaugat un alt fişier:

Sintaxa acestui fişier este:

<?xml version="1.0"?>
<SECURITY>
<HTTP TRACE="ON" />
<AUTH TYPE="Basic" />
<ACTION name="Host" value="localhost">ALLOW</ACTION>
<USER NAME="God" PASS="godpassword" READ="TRUE" WRITE="TRUE" BROWSE="TRUE" EXECUTE="TRUE" />
<!--This is the setting for the user-->
<USER NAME="Guest" PASS="" READ="TRUE" BROWSE="TRUE"/>
<!--The guest user-->
<USER NAME="Tim" PASS="Tom" READ="TRUE" EXECUTE="TRUE" BROWSE="TRUE"/>
<!--Another user-->
<ITEM FILE="file1" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE">
<ACTION name="Host" value="remote">DENY</ACTION>
<!--Define a file and its permissions-->
<USER NAME="Jim" PASS="Morrison" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE" />
<!--Define the user permissions for the file-->
<USER NAME="Bob" PASS="Marley" READ="TRUE" WRITE="FALSE" EXECUTE="TRUE" />
<ACTION name="Host" value="localhost">DENY</ACTION>
<!--Another user permissions for the file-->
</ITEM>
<--End of the file block-->
</SECURITY>

Rândul <HTTP TRACE="ON" /> este opţional şi este descris în secţiunea Gazde Virtuale a acestui ghid. Urmaţi acel link pentru mai multe detalii.

Rândul <AUTH TYPE="Basic" /> este opţional şi selectează schema de autentificare care va fi folosită pentru site-ul dvs. Valoarea implicită este "Basic", însă se poate folosi "Digest" şi atunci rândul va arăta aşa:

<AUTH TYPE="Digest" />
dacă răndul lipseşte din fişier valoarea implicită "Basic" va fi folosită.
Schema "Digest" este mai sigură deoarece parolele nu sunt transferate in text prin reţea.

Se verifică mai întâi dacă utilizatorul are drepturi aici:
<USER NAME="Bob" PASS="Marley" READ="TRUE" WRITE="FALSE" EXECUTE="TRUE" />

Dacă nu sunt drepturi definite, se încearcă folosirea drepturilor definite valabile pentru fişier:
<ITEM FILE="file1" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE">

Dacă nici drepturile pentru fişier nu sunt definite, se folosesc setările generale valabile pentru utilizator:

<USER NAME="Guest" PASS="" READ="TRUE" BROWSE="TRUE"/>

Dacă nici drepturile pentru utilizator nu sunt definite, o eroare va fi trimisă utilizatorului.



Rândurile <ACTION> de mai jos definesc dacă accesul la o resursă este permis sau nu, în funcţie de condiţii. De exemplu, rândul


<ACTION name="Host" value="remote">DENY</ACTION>

înseamnă că fişierul "file1" nu poate fi accesat dacă calculatorul "Host" de pe care se încearcă accesul este "remote". Dacă însă sunt alte setări pentru acest fişier pentru alte calculatoare, ca de exemplu "localhost", acesta poate fi accesat, cu excepţia utilizatorului Bob, din cauza liniei de mai jos:

<ACTION name="Host" value="localhost">DENY</ACTION>


Dacă fişierul "security" nu există, un fişier implicit va fi folosit. Acesta se găseşte în directorul system.
