FASTCGI

FastCGI este o extensie a CGI, independentă de limbajul utilizat, scalabilă şi non proprietară care asigură o performanţă ridicată şi eliminând limitările specifice diverselor API-uri ale serverului. Dacă folosiţi FastCGI, nu vor fi creeate procese separate pentru fiecare cerere a scriptului. Comunicarea cu procesul MyServer are loc prin intermediul sockeţilor.

Pentru a folosi FastCGI în MyServer, configuraţi MIME Type pentru scriptul respectiv să ruleze ca FastCGI (să fie RUNFASTCGI).

Citiţi secţiunea PHP despre cum să folosiţi PHP+FastCGI.

Ca şi la CGI, puteţi creea un director cgi-bin în directorul web în MyServer după cum urmează şi să puneţi în el toate aplicaţiile:



Directorul cgi-bin este fiu direct al directorului web care este selectat.



Totuşi, scripturile FastCGI se pot afla oriunde atâta timp cât sunt recunoscute de serverul de web prin extensia lor.


Servere FastCGI la distanţă

MyServer poate folosi un server FastCGI la distanţă (aflat pe alt calculator). Următorul tip MIME foloseşte o conexiune externă:
<MIMETYPE>
<EXT>php</EXT>
<MIME>text/html</MIME>
<CMD>RUNFASTCGI</CMD>

<MANAGER>@host:port</MANAGER>
</MIMETYPE>
Definiţia pentru un manager la distantă trebuie să înceapă cu @.
Dacă configuraţi tipurile MIME folosind aplicaţia Control center, puteţi adăuga un server la distanţă adăugând o linie cu următoarea formă
@HOST:PORT pentru calea către serverul FastCGI.
