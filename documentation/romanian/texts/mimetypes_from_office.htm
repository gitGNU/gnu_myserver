TIPURI MIME



Puteţi modifica tipurile MIME folosite de către MyServer modificând fişierul "MIMEtypes.xml". Următoarele rânduri sunt un exemplu de cum ar putea arata fişierul MIMEtypes.xml:
<MIMETYPES>
-
    <MIMETYPE>
-
    <MIMETYPE defaultAction="DENY">
<EXT>htm</EXT>
<MIME>text/html</MIME>
<CMD>SEND</CMD>
<ACTION name="Host" value="localhost">ALLOW</ACTION>
<FILTER>gzip</FILTER>
<MANAGER>NONE</MANAGER>
</MIMETYPE>


Fiecare bloc descrie un tip MIME diferit.

<EXT> este tipul fişierului

<MIME> este tipul MIME.

<CMD> este felul în care va fi trimis fişierul clientului.

<ACTION name="Host" value="localhost">ALLOW</ACTION> va permite accesul la tipul MIME dorit dacă Host este "localhost". În linia <MIMETYPE defaultAction="DENY"> vom preciza că tipul nostru MIME va interzice accesul gazdelor neconfigurate explicit cu un tag ACTION corespunzător (deci, acest tag trebuie sa existe configurari ca sa permită accesul de la localhost). Properietatea
defaultAction este opţională (dacă este omisă, aceasta are valoarea implicită "ALLOW"). Deci, în exemplul de mai sus, fişierele HTML vor fi accesate doar folosind hostname-ul "localhost".

Tagul <ACTION> este opţional, dacă nu are valoarea "DENY" pentru defaultAction în tipul MIME.

<FILTER> defineşte un filtru pentru respectivul tip MIME. Datele vor fi transferate printr-un filtru înainte de a fi trimise utilizatorului. În exemplul de mai sus, datele vor fi compresate cu gzip şi apoi trimise utilizatorului. Asta înseamnă că GZip lucrează ca un filtru pentru MyServer. Se poate defini ca Gzip să ruleze în mod automat pentru fiecare fişier a cărui dimensiune depăşeşte o anumită valoare setând <GZIP_THRESHOLD>, în fişierul myserver.xml. Evident că fişierele astfel selectate vor fi trimise compresat doar dacă clientul poate decompresa gzip - altfel acesta ar vedea doar o înşiruire de caractere ciudate în locul fişierului cerut.

În general filtrele pot fi aplicate şi pe datele dinamice - cum ar fi CGI, ISAPI, FastCGI - însă acestea nu trebuie sa modifice conţinutul paginii care urmează să fie afişată clientului.

<MANAGER> descrie comanda<CMD>.



Următoarele valori pentru felul în care sunt trimise fişierele clientului sunt valabile în secţiunea <MANAGER> :

  1. SEND - Trimite fişierul aşa cum este.  Această comandă este folosită pentru fişiere cu extensia .html şi .jpg care nu necesită o interpretare din partea serverului.
  2. RUNCGI - Executa un script CGI indicat în path.  Este folosit pentru fişiere cu extensia .php şi .pl care necesita procesare înainte de a fi trimise clientului.
  3. RUNFASTCGI - Execută un script Fastcgi.  Acest mod de rulare necesită un parser de fastcgi să fie downloadat (vezi pagina despre PHP).  Este folosit tot pentru extensiile .php şi .pl
  4. EXECUTEFASTCGI - Acest mod ruleaza de asemenea un script fastcgi însă spre deosebire de RUNFASTCGI nu are nevoie de un server care să ruleze scriptul ci înglobează funcţionalitatea serverului.
  5. RUNMSCGI- Rulează un script MSCGI. Aceasta librărie este disponibilă doar pentru MyServer. Este folositoare pentru a rula repede şi eficient scripturi CGI.
  6. EXECUTE -Tratează fişierul ca un executabil şi trimite outputul clientului. 
  7. SENDLINK - Trimite clientului fişierul pointat de linkul respectiv.
  8. EXECUTEWINCGI - Trimite fişierul ca un WinCGI
  9. RUNISAPI - Execută un script ISAPI.  
  10. EXECUTEISAPI - Execută un script ISAPI însă nu necesită un server ISAPI pentru al executa. Acest tip înglobează funcţionalitatea serverului.
Majoritatea tipurilor MIME sunt deja configurate în fişierul .xml.  Dacă aveţi totuşi nevoie să adăugaţi alte tipuri MIME folosiţi codul de mai sus ca exemplu.
