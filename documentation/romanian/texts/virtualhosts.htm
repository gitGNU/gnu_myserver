GAZDE VIRTUALE


Teoretic, gazdele virtuale permit găzduirea unui număr infinit de siteuri pe o singură masina(calculator). De exemplu http://mypc.net este diferit de http://sub.mypc.net. Pentru a defini gazdele virtuale în MyServer deschideţi fişierul virtualhosts.xml într-un editor de text (cum ar fi notepad pe Windows sau GNU Vim/Emacs pe GNU/Linux). Un exemplu de configurare pentru gazde virtuale arată aşa:

<VHOSTS>
-
    <VHOST>
<NAME>Every connection</NAME>
<PORT>80</PORT>
<IP>127.0.0.1</IP>
<PROTOCOL>HTTP</PROTOCOL>
<DOCROOT>web</DOCROOT>
<SYSFOLDER>system</SYSFOLDER>
<HOST>www.mysite.com</HOST>
<ACCESSLOG>logs/myServer.log</ACCESSLOG>
<WARNINGLOG>logs/myServer.err</WARNINGLOG>
<ALLOW_CGI>NO</ALLOW_CGI>
</VHOST>
-
</VHOSTS>




<NAME> indică administratorului serverului ce gazdă virtuală este configurată ca spre exemplu conexiunea internă, externă sau oricare dintre conexiuni. 

<PORT> portul folosit pentru conexiuni
<IP> este opţional. Reprezintă adresa IP a interfeţei pe care va răspunde gazda virtuală. Se pot indica (este opţional) useRegex="YES" adrese ca expresii regulate, nu doar adrese prestabilite. Acestea sunt de obicei folosite pentru reţelele cu adresare dinamică.
<PROTOCOL> este protocolul folosit de către gazdă
<DOCROOT> este directorul unde sunt găsite fişierele siteului
<SYSFOLDER> directorul system
<HOST> este numele gazdei virtuale. La fel, se poate folosi (este opţional) useRegex="YES" adresa ca expresie regulată, nu doar ca valoare prestabilită.
<ACCESSLOG> este jurnalierul de accesări al acestei gazde virtuale
<WARNINGLOG> este jurmalierul de erori al acestei gazde virtuale.
<ALLOW_CGI> Această opţiune interzice accesul la CGI a gazdei virtual. Acest rând poate exista de asemenea în myserver.xml, ca configurare globală, însă dacă este inclus în virtualhosts.xml, acesta are prioritate. ALLOW_CGI poate apărea ca ALLOW_FASTCGI, ALLOW_ISAPI, ALLOW_MSCGI , ALLOW_WINCGI, ALLOW_SEND_LINK, ALLOW_EXTERNAL_COMMANDS, ALLOW_SEND_FILE, fiecare permiţând/interzicând folosirea respectivului handler HTTP.


Fişierele jurnalier pot avea câteva extraopţiuni:
<ACCESSLOG type="combined">logs/myServerHTTP.log</ACCESSLOG>
Cu această opţiune, jurnalele vor conţine două noi câmpuri: HTTP Referer şi HTTP User Agent.
Folosind cycle="yes" permiteţi scrierea ciclică a jurnalului. Când un fişier jurnal scris ciclic ajunge la dimensiunea maximă conţinutul acestuia se scrie într-un nou fişier iar jurnalierul iniţial este golit(recreat). Implicit jurnalierele folosesc compresia gzip pentru a economisi spaţiu, însă dacă nu doriţi să folosiţi gzip specificaţi cycle_gzip="no".

MyServer oferă posibilitatea de a activa comanda HTTP TRACE pentru fiecare gazdă virtuală. Tot ce trebui să faceţi e sa editaţi fişierul security.xml în directorul rădăcină al gazdei dvs. virtuale (în rădăcină nu în vre-un subdirector! Dacă un astfel de fişier nu există puteţi creea unul nou în formatul descris în secţiunea Securitate a documentaţiei) şi adăugaţi parametrul

<HTTP TRACE="ON" />

Acest rând activează comanda HTTP TRACE. În mod implicit este inactivă (deci ştergeţi acest rândul dacă doriţi să dezactivaţi această comandă)



MyServer oferă facilitatea de a naviga pe symlinkuri pe sistemele de operare bazate pe Unix(de ex.: Linux, *BSD, GNU/HURD, Solaris, etc.).
Puteţi adăuga facilitatea de navigare pe symlinkuri pentru fiecare gazdă virtuală în fişierul virtualhosts.xml.
<FOLLOW_LINKS>YES</FOLLOW_LINKS>

Implicit este: <FOLLOW_LINKS>NO</FOLLOW_LINKS>
