PAGES D'ERREUR


Vous pouvez modifier les pages HTML qui sont retournées lorsqu'une erreur survient.

Par défaut, les fichiers utilisés sont contenus dans le répertoire system et peuvent donc être remplacés par vos propres fichiers. Plusieurs Hôtes Virtuels peuvent utiliser differents fichiers d'erreur.

Les pages ne doivent contenir aucune références à d'autres fichiers sur répertoire system.
Chaque requête vers /sys renverra au repertoire system. Par conséquent, une requete vers http://sys/file.html retournera le fichier file.html contenu dans le répertoire system de l'hôte virtuel.


La célèbre erreur 404 arrive lorsqu'un fichier ne peut pas être trouvé sur le serveur.

Pour définir une page d'erreur personnalisée pour un hote virtuel, vous pouvez ajouter la ligne suivante dans le fichier sécurité de l'hôte virtuel principal. En définissant une page d'erreur personnalisée, le serveur cherchera dans le répertoire web plutôt que system. Si aucun fichier personnalisé n'est renseigné, le serveur chercher le fichier x.html dans le répertoire system, où x est le numéro d'erreur (par exemple: 404.html). Un fichier sécurité de l'hôte virtuel principal est un fichier de sécurité placé dans le répertoire principal de l'hôte virtuel. Si l'erreur n'est pas indiquée alors la page par défaut du répertoire system sera utilisée. Si la page n'existe pas dans ce repertoire, l'erreur sera envoyée via un message d'erreur HTTP.
<ERROR FILE="error404.html" ID="404" />



Pour plus d'information sur les pages d'erreurs, consultez la documentation du RFC2068 sur www.w3c.org.
