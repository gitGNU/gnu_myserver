TYPES MIME



Vous pouvez modifier les types MIME utilisés par MyServer en modifiant le fichier "MIMEtypes.xml". Voici un exemple de fichier .xml:
<MIMETYPES>
-
<MIMETYPE>
-
<MIMETYPE defaultAction="DENY">
<EXT>htm</EXT>
<MIME>text/html</MIME>
<CMD>SEND</CMD>
<ACTION name="Host" value="localhost">ALLOW</ACTION>
<FILTER>gzip</FILTER>
<MANAGER>NONE</MANAGER>
</MIMETYPE>


Chaque bloque décrit un type MIME différent.

<EXT> est l'extension du fichier

<MIME> est le type Type.

<CMD> est le type d'envoi au client.

<ACTION name="Host" value="localhost">ALLOW</ACTION> permettra l'accès au type MIME désiré si l'hôte est "localhost". La ligne <MIMETYPE defaultAction="DENY"> indique que notre type MIME refusera l'accès à tout hôte virtuel ne possédant pas de ligne ACTION (i.e. il devra exister dans la configuration). De plus, la proprieté
defaultAction
est optionelle (il vaut "ALLOW" s'il est omis). Ainsi, dans l'exemple ci dessus, les ficheir HTML ne seront accessibles que sur l'hôte "localhost".

La balse <ACTION> est optionelle, à moins d'indiquer "DENY" comme defaultAction de votre type MIME.

<FILTER> défini un filtre pour ce type MIME. Les données seront filtrées avant d'être envoyés à l'utilisateur. Dans l'exemple ci dessus, les données seront envoyées à l'utilisateur compressés en gzip. Cela veut dire que gzip fonctionne comme un filtre dans MyServer. Vous pouvez définir gzip pour être lancé à chaque fois qu'une certaine taille de données doit être envoyée, en spécifiant l'option <GZIP_THRESHOLD> dans votre fichier myserver.xml. Pour des raison évidentes, les données seront envoyées en gzip seulement si le client peut recevoir de tels fichiers - sans quoi il recevrait d'étranges caractères à la place du fichier désiré.

En règle général, les filtres peuvent également être appliqués à des données dynamiques - comme CGI, ISAPI, FastCGI - mais ils ne peuvent pas modifier le contenu des pages envoyées à l'utilisateur.

<MANAGER> décrit la commande <CMD>



Les valeurs suivantes d'envoi de fichier au client sont valables dans la section <MANAGER>:

  1. SEND - Envoi le fichier en texte brut. Cette commande est utilisée pour des extensions comme .html ou .jpg, qui n'ont pas besoin d'intervention externe au navigateur.
  2. RUNCGI - Execute le script CGI spécifié par le chemin. Il est utilisé pour des extensions comme .php ou .pl qui nécessite d'être traités avant d'être envoyé au client.
  3. RUNFASTCGI - Execute le script FastCGI. Ce type nécessite le téléchargement d'un parse fastcgi (voir la page PHP). Il est également utilisé pour des extensions comme .php et .pl
  4. EXECUTEFASTCGI - Execute également le script FastCGI, mais contrairement à RUNFASTCGI ci dessus, il n'a pas besoin d'un serveur pour executer le script, mais fonctionne comme un propre serveur.
  5. RUNMSCGI - Execute un script MSCGI. Cette bibliothèque n'est utilisé que par myServer. Il permet la création de CGIs rapides et efficaces.
  6. EXECUTE - Traite le fichier comme un executable et retourne le résultat au client. 
  7. SENDLINK - Retourne le fichier vers lequel pointe le lien.
  8. EXECUTEWINCGI - Envoie le fichier comme WinCGI
  9. RUNISAPI - Execute un script ISAPI.  
  10. EXECUTEISAPI - Execute également un script ISAPI mais sans besoin d'un serveur. Agit comme son propre serveur à la place.
La plupart des applications sont déjà supportés dans le fichier .xml. Si vous voulez ajouter un type MIME, rajoutez le dans le fichier en utilisant l'exemple.
