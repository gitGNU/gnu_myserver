CONFIGURATION D'UN SERVEUR DE BASE

Dans le fichier myserver.xml, vous pouvez définir la page par défaut pour vos sites hébergés avec MyServer.

Les lignes suivantes existent déjà dans votre fichier de configuration:

<DEFAULT_FILENAME0>default.html</DEFAULT_FILENAME0>
<DEFAULT_FILENAME1>default.php</DEFAULT_FILENAME1>
<DEFAULT_FILENAME2>default.php3</DEFAULT_FILENAME2>
<DEFAULT_FILENAME3>index.html</DEFAULT_FILENAME3>
<DEFAULT_FILENAME4>index.php</DEFAULT_FILENAME4>
<DEFAULT_FILENAME5>index.php3</DEFAULT_FILENAME5>
<DEFAULT_FILENAME6>default.htm</DEFAULT_FILENAME6>


Ainsi, si vous avez un Hôte Virtuel http://mypc.net, la page qui s'affichera en se rendant à cette adresse dans votre navigateur sera l'un des fichiers ci dessus. Vous pouvez ajouter d'autres lignes ou les changer si vous avez besoin d'autres noms. Le même fonctionnement est utilisé pour chaque répertoire et non pas juste pour la racine.



Dans la section Erreurs, vous avez appris à créer vos propres pages d'erreur. Mais vous pouvez aussi désactiver ces pages d'erreurs personnalisées en changeant la ligne ci dessous:
<USE_ERROR_FILE>YES</USE_ERROR_FILE>
Ainsi, en mettant la valeur à NO comme ci dessous,
<USE_ERROR_FILE>NO</USE_ERROR_FILE>
MyServer enverra (par défaut) une page d'erreur à votre navigateur contenant le code erreur. Mais vous pouvez y ajouter le contenu de l'erreur en ajoutant la ligne:
<ERRORS_INCLUDE_BODY>YES</ERRORS_INCLUDE_BODY>

Vous pouvez changer le style de la page de navigation de fichiers (utilisé lorsque vous n'avez aucune page de défaut dans le répertoire, tel que décrit dans la section Navigation de fichiers). La ligne ci dessous, dans myserver.xml, indique le fichier CSS utilisé pour configurer le style des pages de navigation:
<BROWSEFOLDER_CSS>system/folderstyle.css</BROWSEFOLDER_CSS>
 
 
Les scripts CGI peuvent obtenir l'adresse courriel de l'administrateur (au moyen de la variable d'environnement SERVER_ADMIN). Il peut donc être important de définir proprement cette adresse, en modifiant la valeur de la ligne suivante dans myserver.xml:
<SERVER_ADMIN>administrator@localhost</SERVER_ADMIN>
MyServer offre la possibilité de suivre les liens symboliques des systèmes d'exploitation de type Unix (i.e.: Linux, *BSD, GNU/HURD, Solaris, etc.).
Vous pouvez ajouter l'option de liens symboliques dans le fichier myserver.xml.
<FOLLOW_LINKS>YES</FOLLOW_LINKS>

La valeur utilisée par défaut dans MyServer est: <FOLLOW_LINKS>NO</FOLLOW_LINKS>.
