SERVEUR WEB

Qu'est ce qu'un serveur web?

Un serveur web est un programme permettant aux utilisateurs de partagers des documents sur Internet ou sur un réseau local.

Vous avez uniquement besoin d'un navigateur pour vous connecter au serveur web.


Plusieurs clients se connectant à un serveur web.

Cette image montre comment des clients variés peuvent accéder à votre serveur au même moment. La requête arrive sur votre serveur grâce au réseau. Une réponse est générée par le serveyr et renvoyée au client. L'évolution d'Internet a fait du modèle Client-Serveur la façon la plus simple de partager rapidement des données, offrant ainsi la possibilité à chaque utilisateur de choisir ce qu'il ou elle veut partager avec les autres utilisateurs.

Comment les fichiers sont-ils partagés?

Grâce au niveau le plus basique, le serveur web partage des documents HTML et des images. Ces documents et images sont de simples fichiers statiques correspondant aux fichiers que vous voyez lorsque vous accédez à un site web. Ces fichiers sont transférés au moyen du protocole HTTP (HyperText Transfer Protocol).

Si du contenu dynamique est nécessaire, par exemple pour faire fonctionner un site d'e-commerce, différents protocoles sont utilisés pour la communication du serveur web avec un processus externe. On trouve parmis ces protocoles : Common Gateway Interface (CGI), ISAPI, FastCGI ou bien encore WinCGI.
HTTP n'est pas un protocole sécurisé, car les données transmises ne sont pas cryptés. Le cryptage reste possible pour des transmissions sécurisées grâce à HTTPS (HyperText Transfer Procol Secure). Plus d'informations sur HTTPS peuvent être trouvées ici.

Traduction de chemin

Les serveurs web traduisent un chemin d'une URI en une ressource système locale. Le chemin URI spécifié par le client est relatif au répertoire racine du serveur web (Hôtes virtuels). Par exemple, le client peut demander l'accès au fichier /mydirectory/myfile avec la commande suivante:

GET /mydirectory/myfile HTTP/1.1
Host: www.host.com

Le serveur rajoute au répertoire racine de l'hôte virtuel courrant le nom du fichier. Si le répertoire racine est /var/www, alors la ressource retournée sera /var/www/mydirectory/myfile, en fonction des spécifications de son type MIME.
