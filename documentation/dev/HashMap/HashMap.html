HashMap<KeyType, ValueType>



Description:

HashMap is a template class that efficiently associates objects of type KeyType with objects of type ValueType.



Header file:

hash_map.h



Template parameters:

KeyType represents the type of the keys.

Interprets Strings as STL strings, char* as C-like(null terminated) strings, void* as pointers and everything else as the respective type.

ValueType represents the type of the values.

Interprets everything as their respective type.



Types:

Iterator

Iterator()

This doesn't do anything, no arguments.

bool Operator==

Checks if two Iterators are equal.

Complexity: O(1), constant time.

bool Operator!=

Checks if two Iterators are not equal.

Complexity: O(1), constant time.

ValueType Operator*

Returns the value that Iterator is pointing to.

Complexity: O(1), constant time.

Iterator Operator++(prefix)

Points to the next element on the HashMap and returns an Iterator to it.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one low enough, will degrade the performance to O(N).

Iterator Operator++(sufix)

Returns an Iterator to the current element on the HashMap and points the Iterator to the next element on the HashMap.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one low enough, will degrade the performance to O(N).

Iterator Operator--(prefix)

Points to the previous element on the HashMap and returns an Iterator to it.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one low enough, will degrade the performance to O(N).

Iterator Operator--(sufix)

Returns an Iterator to the current element on the HashMap and points the Iterator to the previous element on the HashMap.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one low enough, will degrade the performance to O(N).

KeyType getKey()

Returns the key the iterator is currently pointing at.

Complexity: O(1), constant time.



Members:

HashMap()

Creates an HashMap object with the standard initial size of 16, which is also the minimal size, and astandard loadFactor of 0.75. The loadFactor is the percentage of space the HashMap must have filled before it increases it's size. In this case it will only increase it's size when it has more then 12 elements out of the 16.

Complexity: O(N), where N is the size of the object.

HashMap(int size)

Same as HashMap() except it specifies the initial size for the HashMap object.

Complexity: O(N), where N is the size of the object.

HashMap(const float loadFactor)

Same as HashMap() except it specifies the loadFactor for the HashMap object.Complexity: O(N), where N is the size of the object.

HashMap(int size, const float loadFactor)

Same as HashMap() except it specifies both the initial size and the loadFactor for the HashMap object.

Complexity: O(N), where N is the size of the object.

bool empty(void)

Returns true if HashMap object is empty, otherwise false.

Complexity: O(1) constant time.

void clear(void)

Clears all elements.

Complexity: O(1), constant time.

int size(void)

Returns the amount of elements currently in the HashMap object.

Complexity: O(1), constant time.

Iterator begin(void)

Returns an Iterator pointing to the first element on the HashMap object. Note that the order of the elements in the HashMap object has no relation to the order by which the elements were inserted or their size or their numerical value.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one low enough, will degrade the performance to O(N).

Iterator back(void)

Returns an Iterator pointing to the last element on the HashMap object. Note that the order of the elements in the HashMap object has no relation to the order by which the elements were inserted, their size or their numerical value.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one low enough, will degrade the performance to O(N).

Iterator end(void)

Returns an Iterator pointing to the element after the last element on the HashMap object. Note that the order of the elements in the HashMap object has no relation to the order by which the elements were inserted, their size or their numerical value.

Complexity: O(1), constant time.

bool containsKey(const KeyType& key)

Returns true if the HashMap object contains key, otherwise false.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one high enough, will degrade the performance to O(N).

ValueType get(const KeyType& key)

Returns the value associated with key, if there is none returns zero. Note that if the value associated with key IS zero, there is no way to know, using this method if there was or not a value associated with key, to be sure you can either avoid using zeros or use the containsKey method or the getI method.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one high enough, will degrade the performance to O(N).

Iterator getI(const KeyType& key)

Returns an Iterator object pointing to the value associated with key.

Complexity: O(1) ,constant time. Note that chosing a poor loadFactor, specifically one high enough, will degrade the performance to O(N).

ValueType put(const KeyType& key, const ValueType& value)

Inserts a new key in the HashMap object and associates it with value. Returns the previously associated with key if there was one, otherwise returns zero. Note that if the value associated with key IS zero, there is no way to know, using this method if there was or not a value associated with key, to be sure you can either avoid using zeros or use the containsKey method or the getI method.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one high enough, will degrade the performance to O(N).

ValueType remove(const KeyType& key)

Removes key from the HashMap object and returns the value associated with it. If key isn't found, returns zero. Note that if the value associated with key IS zero, there is no way to know, using this method if there was or not a value associated with key, to be sure you can either avoid using zeros or use the containsKey method or the getI method.

Complexity: O(1), constant time. Note that chosing a poor loadFactor, specifically one high enough, will degrade the performance to O(N).

ValueType remove(Iterator iter)

Removes the element iter is pointing at from the HashMap object and returns the value associated with it.

Complexity: O(1), constant time.
