CONFIGURAÇÃO BÁSICA DO SERVIDOR

No arquivo myserver.xml você pode setar a página default para seus websites hospedados no MyServer.

As linhas abaixo já existem no seu arquivo de configuração:

    <DEFAULT_FILENAME0>default.html</DEFAULT_FILENAME0>
    <DEFAULT_FILENAME1>default.php</DEFAULT_FILENAME1>
    <DEFAULT_FILENAME2>default.php3</DEFAULT_FILENAME2>
    <DEFAULT_FILENAME3>index.html</DEFAULT_FILENAME3>
    <DEFAULT_FILENAME4>index.php</DEFAULT_FILENAME4>
    <DEFAULT_FILENAME5>index.php3</DEFAULT_FILENAME5>
    <DEFAULT_FILENAME6>default.htm</DEFAULT_FILENAME6>



Assim, se você tem um Virtual Host http://mypc.net, a página que vai aparecer quando você acessar aquele endereço no browser, será um dos arquivos acima. Você pode adicionar mais linhas, ou mudar qualquer uma delas, se precisar de um nome de arquivo diferente. O mesmo método é usado para cada diretório, e não só o diretório raiz do site.


Na seção de Errors você aprendeu como criar suas próprias páginas de erro. Mas você também pode desabilitar as páginas personalizadas, mudando a linha abaixo:
<USE_ERRORS_FILES>YES</USE_ERRORS_FILES>
Desse modo, se você setar o valor para NO, como nessa linha
<USE_ERRORS_FILES>NO</USE_ERRORS_FILES>
você terá mensagens de TEXTO ao invés de páginas HTML para os erros.
Você pode mudar o layout da página de Navegação de Pastas (usada quando você não tem uma página default, como descrito na seção Navegação de Pastas). A linha abaixo, no myserver.xml, aponta para o arquivo CSS usado para configurar o layout de tal página:
<BROWSEFOLDER_CSS>system/folderstyle.css</BROWSEFOLDER_CSS>

Scripts CGI podem obter o E-mail do Administrador do Servidor e mostrá-lo ao usuário. Então é importante que você defina esse endereço corretamente, setando o valor na linha abaixo, no myserver.xml:
<SERVER_ADMIN>administrator@localhost</SERVER_ADMIN>
