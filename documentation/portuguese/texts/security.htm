CONTROLANDO O ACESSO

Para controlar o acesso a um arquivo, você tem que colocar o arquivo security na pasta que gostaria de proteger.

A sintaxe desse arquivo é:
<?xml version="1.0"?>
<SECURITY>
<HTTP TRACE="ON" />
<AUTH TYPE="Basic" />
<USER NAME="God" PASS="godpassword" READ="TRUE" WRITE="TRUE" BROWSE="TRUE" EXECUTE="TRUE" />
<!--Aqui é a configuração para o usuário-->
<USER NAME="Guest" PASS="" READ="TRUE" BROWSE="TRUE"/>
<!--O usuário guest-->
<USER NAME="Tim" PASS="Tom" READ="TRUE" EXECUTE="TRUE" BROWSE="TRUE"/>
<!--Outro usuário-->
<ITEM FILE="file1" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE">
<!--Define um arquivo e suas permissões-->
<USER NAME="Jim" PASS="Morrison" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE" /> 
<!--Define as permissões de usuário para o arquivo-->
<USER NAME="Bob" PASS="Marley" READ="TRUE" WRITE="FALSE" EXECUTE="TRUE" />
<!--Permissões do outro usuário para o arquivo-->
</ITEM>
<--FIm do bloco de segurança-->
</SECURITY>

A linha <HTTP TRACE="ON" /> é opcional, e está descrita na seção que trata de Virtual Hosts. Por favor, dê uma olhada naquele link, se quiser saber sobre aquela linha.

A linha <AUTH TYPE="Basic" />, opcional, seleciona o esquema de autenticação a ser usada para seu site. O valor default é o "Basic", mas você pode utilizar também o esquema "Digest", ficando desse modo:

<AUTH TYPE="Digest" />
Se a linha não estiver contida no arquivo, será usada a autenticação "Basic".
O esquema "Digest" é mais seguro, pois a senhas não trafegam em aberto, pela rede.

O programa primeiro verifica se o usuário tem permissões no bloco do arquivo:

<USER NAME="Bob" PASS="Marley" READ="TRUE" WRITE="FALSE" EXECUTE="TRUE" />

Se não houver permissões definidas, o sistema tenta usar as permissões que estão definidas para o arquivo:

<ITEM FILE="file1" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE">

Se não pude encontrar nenhuma permissão para o arquivo, o sistema tenta usar as configurações gerais para o usuário:

<USER NAME="Guest" PASS="" READ="TRUE" BROWSE="TRUE"/>

Se o usuário não estiver definido, o sistema vai enviar um erro para o usuário. Se o arquivo de segurança não estiver presente, um arquivo de segurança default será usado pelo sistema. Esse arquivo default está na pasta system.
