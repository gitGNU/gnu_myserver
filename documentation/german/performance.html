LEISTUNGS-TUNING

Hier sehen Sie einige Optionen, die Sie einstellen können, um die Leistung Ihres Webservers zu verbessern.

Threads Tuning:
In der Dateimyserver.xml können Sie die Anzahl der Threads, die MyServer benutzen wird einstellen. In den folgenden Zeilen konfigurieren Sie
2 einstellbare Parameter A und B, der Gleichung N=A+CB, bei der N die Anzahl der zu benutzenden Threads ist und C die Anzahl der im Computer
verfügbaren Prozessoren ist:
    <NTHREADS_A>10</NTHREADS_A>
    <NTHREADS_B>10</NTHREADS_B>
In dem Sie, den für Sie zutreffenden Wert (Standard=10) in den oben aufgelisteten Variablen verwenden, ändern Sie den Wert von N.



Buffer Größe:
Sie können die Buffer Größe, die von MyServer verwendet wird ändern, indem Sie folgende Zeile in der Datei myserver.xml ändern:
    <BUFFER_SIZE>102400</BUFFER_SIZE>
Sie brauchen bloß den standard Wert 102400 in die von Ihnen gewünsche Göße zu ändern. Berücksichtigen Sie, dass der Buffer doppelt so groß sein wird,
wie der Wert (in Bytes), den Sie hier eingestellt haben, multipliziert mit der Anzahl der laufenden Threads.


Verbindungs Timeout:
In der myserver.xml können Sie den Timeout für Client Verbindungen zu Ihrem Server einstellen, wie folgende Zeile zeigt:
    <CONNECTION_TIMEOUT>60</CONNECTION_TIMEOUT>
Die Zeit Einheit ist in Sekunden. In der Zeile sieht man, dass die Verbindung zu einem Client, der auf Ihre Seite länger als 60 Sekunden nicht mehr
zugreift geschlossen wird. Falls Sie es bevorzugen keine HTTP Keep-Alive Vebindungen zu benutzen, setzen Sie den Wert auf 0.


Max Verbindungen:
Sie können die Standard-Anzahl der Verbindungen, die Ihr Webserver akzeptiert ändern, in dem Sie folgenden Zeile in der myserver.xml ändern:
    <MAX_CONNECTIONS>0</MAX_CONNECTIONS>
Durch den Standardwert 0 akzeptiert Ihr Server unendlich viele Verbindungen. Jeder andere Wert beschränkt die Anzahl der Benutzer, die auf Ihren Server zugreifen werden können.
