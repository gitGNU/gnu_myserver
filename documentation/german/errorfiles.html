FEHLERBEHANDLUNG



Sie können HTML Seiten erstellen, die gesendet werden, wenn ein Fehler auftritt.

Die standard Dateien sind im system Verzeichnis, die vom Host genutzt werden und durch Ihre eigenen Dateien erstezt werden können. Unterschiedliche virtuelle Hosts können verschiedene Error Dateien benutzen.

Die Seiten müssen nicht Referenzen auf andere Dokumente im system Verzeichnis erhalten.






Der berühmete 404 Fehler tritt auf, wenn eine Datei nicht auf dem Server gefunden werden kann.


Um eine eigene Fehlerseite für einen virtuellen Host zu erstellen, fügen Sie folgende Zeile in die primäre virtuelle Host security Datei. Eine primäre security Datei für einen virtuellen Hosts ist die security Datei, die sich im Hauptverzeichnis des virtuellen Hosts befindet.

<ERROR FILE="error404.html" ID="404" />



Mehr Informationen zu den Fehlerseiten gibt es im RFC2068 Dokument auf www.w3c.org.
