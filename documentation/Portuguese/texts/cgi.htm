CGI

CGI (Common Gateway Interface) é a forma padrão com que um servidor web passa os dados do usuário para uma aplicação e então retorna informação ao usuário. CGI depende de scipts freqüentemente escritos em linguagens com PHP ou Perl para realizar as ações que são requisitadas pelo usuário. Para acessar o script CGI, o nome da aplicação é indicado no código HTML.

Por exemplo, se você estiver criando um formulário, a URL do script é indicada nos tags FORMs:

<FORM METHOD=POST ACTION=www.myfiles.com/cgi-bin/myprog.pl>

Obs:  Qualquer Linguagem de  Script que consegue fazer 'pipe' para saída pode ser usada - você pode ter scripts em Shell Script, Python, etc. Você só precisa configurar corretamente os MIME-Types correspondentes, e sua linguagem favorita de scripts funcionará no MyServer.

Contudo, scripts CGI podem ser colocados em qualquer lugar, contanto que sua localização seja colocada no código HTML, e que elas sejam mapeadas para ser CGI pela sua extenção de arquivo.

Scripts CGI geram um novo processo para cada request. Isso pode causar alguns problemas de performance, mas assegura alguma segurança de modo que se o novo processo travar, o processo principal do MyServer não é afetado.
