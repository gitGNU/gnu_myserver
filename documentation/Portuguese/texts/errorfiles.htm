PÁGINAS DE ERRO



Você pode modificar as páginas html que são enviadas quando um erro ocorre.

Os arquivos default ficam dentro da pasta system usado pelo host, e podem ser substituídos pelos seus próprios arquivos. Hosts Virtuais diferentes podem usar arquivos de erro diferentes.

As páginas não devem conter quaisquer referências a outros documentos na pasta system.
Cada request será mapeado para ao diretório system. Então, um request enviará o arquivo file.html, contido no diretório system para o host virtual.


O famoso "Erro 404" acontece quando um arquivo não pode ser encontrado no servidor.


Para definir uma página de erro personalizada para um host virtual você pode adicionar a linha como a seguinte no arquivo primário de segurança do virtual host. Definindo um arquivo de erro diferente, o servidor vai olhar na pasta web ao invés da pasta system. Se nenhum arquivo personalizado for usado, o servidor vai procurar o arquivo x.html na pasta system, onde x é o número do erro (por exemplo:404.html). Um arquivo primário de segurança de um host virtual é o arquivo de segurança que está localizado na pasta principal do virtual host. Se esse arquivo não existir também, o erro será comunicado via HTTP.
<ERROR FILE="error404.html" ID="404" />

Para mais informações sobre páginas de erro, consulte o documento RFC2068, que pode ser encontrado em www.w3c.org.
