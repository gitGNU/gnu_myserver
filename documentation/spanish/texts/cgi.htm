CGI

CGI (Common Gateway Interface) es la manera estándard en la que un servidor Web pasa información de un usuario a una aplicación y luego devuelve información al usuario. CGI depende de secuencias de comandos que suelen estar escritos en PHP o Perl para realizar acciones que son solicitadas por el usuario. Para poder acceder a la secuencia de comandos CGI, se indica el nombre de la aplicación en el código HTML.

Por ejemplo, si crea el formulario, se indica la dirección de las secuencias de comandos en la etiqueta de los Formularios:

<FORM METHOD=POST ACTION=www.myfiles.com/cgi-bin/myprog.pl>

Nota: Se puede utilizar cualquier Lenguaje de Automatización que puede entubar la salida, aunque puede tener secuencias de comandos en Shell Script, Python, etc. Sólo necesita configurar el tipo MIME correcto, y su lenguaje de automatización favorito funcionará en MyServer.

Para mantener todas las secuencias de comandos CGI en un sólo lugar, puede ubicar el directorio cgi-bin en el directorio Web de MyServer como se muestra debajo.



El directorio cgi-bin está directamente debajo de la carpeta Web seleccionada.



Sin embargo, las secuencias de comandos CGI pueden ubicarse en cualquier lugar mientras que su ubicación esté indicada en el código HTML y estén asignadas como CGI a través de su extensión.

Las secuencias de comandos para las cuales un proceso es creado por cada solicitud. Esto puede causar algunos problemas con rendimientos pero asegura cierta seguridad ya que si los nuevos procesos fallan, el proceso principal de MyServer no se corrompe.
