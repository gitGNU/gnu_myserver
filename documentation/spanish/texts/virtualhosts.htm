HOSTS VIRTUALES


En teoría, los hosts virtuales permiten a los usuarios alojar un número infinito de sitios en una sola máquina. Por ejemplo http://mypc.net es diferente a http://sub.mypc.net. Para definir hosts virtuales en MyServer abra el archivo virtualhosts.xml en un editor de texto (como en bloc de notas Windows o GNU Emacs en GNU/Linux). Un archivo de ejemplo de host virtuales se ve así:

<NAME> le da al administrador del servidor una idea de que el host virtual como una Ethernet interna, Ethernet externa o cualquier conexión. 

<PORT> es el puerto usado para aceptar conexiones
<IP> que es opcional. Es la dirección IP para la interfaz de su host virtual en la que trabaja. La opción useRegex="YES" significa que la dirección actuará como una dirección regular, no sólo como un valor fijo. Esto es más útil para redes con direccionamiento IP dinámico.
<PROTOCOL> es el protocolo a ser usado por el host
<DOCROOT> es la carpeta en donde se encuentran los archivos
<SYSFOLDER> es la carpeta de sistema
<HOST> es su nombre de host virtual. La opción useRegex="YES" significa que la dirección actuará como una dirección regular, no sólo como un valor fijo.
<ACCESSLOG> es el registro de acceso a los host virtuales
<WARNINGLOG> es el archivo de registro de errores por el host virtual.
<ALLOW_CGI> Esta opción deniega el acceso al CGI en el host virtual. Esta línea puede existir en myserver.xml también, como una configuración global, pero se incluye en virtualhosts.xml, esta será la que será considerada. ALLOW_CGI puede aparecer como ALLOW_FASTCGI, ALLOW_ISAPI, ALLOW_MSCGI , ALLOW_WINCGI, ALLOW_SEND_LINK, ALLOW_EXTERNAL_COMMANDS, ALLOW_SEND_FILE, cada uno de ellos denegando/permitiendo el controlador de información HTTP apropiado.


Los archivos de errores pueden tener otras opciones:
<ACCESSESLOG type="combined">logs/myServerHTTP.log</ACCESSESLOG>
Con esta opción, los registros tendrán dos campos extras: Remitente HTTP y Agente Usuario HTTP.
Al usar cycle="yes" habilitará el ciclado de registro en el archivo de registros. Cuando un archivo de registro que habilita el "ciclado" alcanza el tamaño mayor para archivos de registros, grabará su contenido a un archivo nuevo y recreará el archivo original. Se usa compresión gzip de los archivos de registro ciclados de manera predeterminada para ahorrar espacio, si no quiere usar gzip especifique cycle_gzip="no".

MyServer le da la opción de habilitar el comando HTTP TRACE para cada host virtual. Sólo necesita editar el archivo security.xml de la raíz de su host virtual (no un subdirectorio, si no existe el archivo, puede crear uno con el formato descripto en la sección Seguridad de esta documentación), y agregue el parámetro:

<HTTP TRACE="ON" />

Una vez que lo hace, habilita el comando HTTP TRACE. Esta predeterminado como deshabilitado (así que sólo elimine la ínea para deshabilitarlo)
