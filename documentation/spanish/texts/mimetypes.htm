TIPOS MIME



Puede modificar los tipos MIME usados por MyServer al modificar el archivo "MIMEtypes.xml". El siguiente, es un ejemplo del archivo .xml:
<MIMETYPES>
-
    <MIMETYPE>
-
    <MIMETYPE defaultAction="DENY">
<EXT>htm</EXT>
<MIME>text/html</MIME>
<CMD>SEND</CMD>
<ACTION name="Host" value="localhost">ALLOW</ACTION>
<FILTER>gzip</FILTER>
<MANAGER>NONE</MANAGER>
</MIMETYPE>


Cada bloque describe un tipo MIME diferente.

<EXT> el tipo de archivo

<MIME> es del tipo Mime.

<CMD> es la manera de enviar el archivo al cliente.

<ACTION name="Host" value="localhost">ALLOW</ACTION> permitirá el acceso a los tipos MIME deseados si el Host es "localhost". En la línea <MIMETYPE defaultAction="DENY"> diremos que nuestro tipo MIME denegará el acceso desde cualquier host que no se haya escrito en la etiqueta ACTION correcta (entonces tendrá que existir en la configuración). También la propiedad
defaultAction es optativa (cuando es omitida, se establece como "ALLOW"). Entonces, en el ejemplo de arriba, los archivos HTML serán accedidos sólo a través del nombre del host "localhost".

La etiqueta <ACTION> es optativa, a menos que se escriba "DENY" para su defaultAction en su tipo MIME.

<FILTER> define un filtro para ese Tipo MIME. La información que pasa a través del filtro antes de ser enviada al usuario. En el ejemplo de arriba, la información sería enviada comprimida con Gzip al usuario. Significa que Gzip trabaja como un filtro en MyServer. Puede definirlo para que se ejecute automáticamente por cada archivo basado en el tamaño, estableciéndolo en <GZIP_THRESHOLD>, en el archivo myserver.xml. Por razones obvias, la información será enviada comprimida sólo si el cliente es compatible con Gzip - de otra manera recibiría un montón de caracteres extraños en vez del sitio requerido.

En general, los filtros también pueden ser aplicados a la información dinámica - como CGI, ISAPI, FastCGI - pero necesita filtros que no modifiquen los contenidos de la página a mostrar al usuario.

<MANAGER> describe el comando <CMD>.



Los siguientes valores para el envío de archivos al cliente son válidos en la sección <MANAGER>:

  1. SEND - Envía el archivo estáticamente como está. Este comando se usa para extensiones de archivos como .html y .jpg que no necesitan una intervención externa desde el servidor Web.
  2. RUNCGI - Ejecuta la secuencia de comandos CGI que está en la ruta de acceso. Se usa para las extensiones como .php y .pl que necesitan ser procesadas antes de enviar cualquier información al cliente.
  3. RUNFASTCGI - Ejecuta la secuencia de comandos Fastcgi. Este tipo necesitará descargar un analizador fastcgi (vea la página PHP). También se usa para extensiones como .php y .pl
  4. EXECUTEFASTCGI - También ejecuta secuencias de comandos Fastcgi, pero a diferencia de RUNFASTCGI arriba, no necesita un servidor para ejecutar una secuencia de comandos, sino que actúa como su propio servidor.
  5. RUNMSCGI - Ejecuta la secuencia de comandos MSCGI. Esta biblioteca se usa solamente para myServer. Se usa para construir CGis rápidos y eficientes.
  6. EXECUTE -Trata el archivo como un ejecutable y envía el resultado al cliente.
  7. SENDLINK - Envía el archivo que contiene el vínculo.
  8. EXECUTEWINCGI - Envía el archivo como WinCGI
  9. RUNISAPI - Ejecuta la secuencia de comandos ISAPI.  
  10. EXECUTEISAPI - Ejecuta la secuencia de comandos ISAPI pero no necesita un servidor para ejecutarse. En cambio actúa como su servidor.
La mayoría de las aplicaciones ya son tratadas en el archivo .xml. Si quiere agregar un tipo MIME, anexe el archivo y use el código como ejemplo.
