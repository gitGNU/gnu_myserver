CONTROLANDO ACCESOS


Para poder controlar el acceso a un archivo, debe agregar seguridad en el directorio en el que le gustaría proteger.

La sintaxis del archivo es:

version="1.0" Versión 1.0 xml XML <?xml version="1.0"?>
<SECURITY>
<HTTP TRACE="ON" />
<AUTH TYPE="Basic" />
<ACTION name="Host" value="localhost">ALLOW</ACTION>
<USER NAME="God" PASS="godpassword" READ="TRUE" WRITE="TRUE" BROWSE="TRUE" EXECUTE="TRUE" />
<!--Esta es la configuración para el usuario-->
<USER NAME="Guest" PASS="" READ="TRUE" BROWSE="TRUE"/>
<!--El usuario invitado-->
<USER NAME="Tim" PASS="Tom" READ="TRUE" EXECUTE="TRUE" BROWSE="TRUE"/>
<!--Otro usuario-->
<ITEM FILE="file1" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE">
<ACTION name="Host" value="remote">DENY</ACTION>
<!--Define un archivo y sus permisos-->
<USER NAME="Jim" PASS="Morrison" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE" />
<!--Define los permisos de usuario para el archivo-->
<USER NAME="Bob" PASS="Marley" READ="TRUE" WRITE="FALSE" EXECUTE="TRUE" />
<ACTION name="Host" value="localhost">DENY</ACTION>
<!--Otros permisos del usuario para el archivo-->
</ITEM>
<--Fin del bloque del archivo-->
</SECURITY>

La línea<HTTP TRACE="ON" /> es opcional, y se describe los Hosts Virtuales en la sección de esta guía. Por favor siga el vínculo si quiere entender sobre esa línea.

La línea <AUTH TYPE="Basic" />, opcional, elige el esquema de autenticación para ser usado en su sitio. Su valor predeterminado es Basic", pero puede usar "Digest", así que se verá de esta manera:

<AUTH TYPE="Digest" />
Si la línea no está en el archivo, se usará el predeterminado, "Basic".
El esquema "Digest" es más seguro, ya que las contraseñas no se transfieren a través de la red.

El motor, primero controla si el usuario tiene permisos en el bloque del archivo:
<USER NAME="Bob" PASS="Marley" READ="TRUE" WRITE="FALSE" EXECUTE="TRUE" />

Si no hay permisos definidos, el motor intenta usar los permisos que están definidos por el archivo:
<ITEM FILE="file1" READ="TRUE" WRITE="FALSE" EXECUTE="FALSE">

Si el motor no encuentra archivos de permisos, usa la configuración general para el usuario:

<USER NAME="Guest" PASS="" READ="TRUE" BROWSE="TRUE"/>

Si el usuario no está definido, el motor enviará un error al usuario.



La línea <ACTION> de arriba define si un recurso le será permitido ser accedido o no, dependiendo de las condiciones. Por ejemplo, la línea


<ACTION name="Host" value="remote">DENY</ACTION>

Significa que el archivo "file1" será denegado si el Host usado para acceder al archivo es "remoto" como hostname. Pero si hay otros mapas del archivo en otros hostnames de Host, por ejemplo "localhost", puede ser accedido, excepto por el usuario Bob, especificado en la línea de la línea de abajo.

ACTION Acción, Acción, Acción, Acción, Acción ACTION Acción, Acción, Acción, Acción, Acción<ACTION name="Host" value="localhost">DENY</ACTION>


Si el archivo de seguridad no está presente, un archivo de seguridad predeterminado será usado por el motor. El archivo de seguridad predeterminado se encuentra en el directorio del sistema.
