PÁGINAS DE ERROR


Puede modificar las páginas html que son enviadas cuando ocurre un error.

Los archivos por defecto están contenidos en el directorio de sistema usado por el host y se puede reemplazar con sus archivos. Diferentes Host Virtuales pueden usar archivos de error diferentes.

Las páginas no deben contener referencias a otros documentos en el directorio del sistema.
Toda solicitud a será asignada al directorio de sistema. Entonces una solicitud a enviará el archivo file.html contenido en el directorio del sistema por el host virtual.


El famoso error 404 ocurre cuando un archivo no puede ser encontrado en el servidor.


Para definir un archivo de páginas de error personalizado para un host virtual puede agregar una línea como la siguiente en el archivo de seguridad del host virtual principal. Al definir un archivo diferente, el servidor buscará en el directorio Web en vez del de sistema. Si no se usan archivos personalizados, el servidor buscará en el archivo x.html en el directorio de sistema, en dónde x es el número de error (por ejemplo: html HTML404.html). Un archivo de seguridad principal para un host virtual es el archivo de seguridad que está ubicado en el directorio principal del host virtual. Si el error no está especificado entonces se usará el predeterminado en el directorio de sistema, si también no existe este archivo será comunicado a través del un mensaje de error HTTP.
