SERVIDOR WEB

¿Qué es un servidor Web?

Un servidor Web es una aplicación, que les permite a los usuarios compartir documentos a través de Internet o una red local.

Para conectarse con el servidor Web necesita un explorador Web.


Muchos clientes se conectan a un servidor Web.

La imagen muestra cuántos clientes pueden estar conectados con un servidor al mismo tiempo. La solicitud llega el servidor a través de la red. El servidor prepara una respuesta y se envía al cliente. La evolución de Internet hizo que el modelo Servidor-Cliente fuera el modo más fácil de compartir información, dándole la posibilidad a cada usuario de decidir que es lo que quiere compartir con otros usuarios.

¿Cómo se comparten los archivos?

Usando el nivel básico, el servidor Web comparte documentos HTML e imágenes. Estos documentos e imágenes son muy simples, estáticos y básicos, que ve cuando accede a un sitio Web. Estos archivos se transfieren por HTTP (HyperText Transfer Protocol).

Si se necesita un contenido dinámico, por ejemplo ejecutar un sitio Web de comercio electrónico, para la comunicación del servidor Web se usa protocolos diferentes con un proceso externo. Algunos de estos protocolos son: Common Gateway Interface (CGI), ISAPI, FastCGI y WinCGI.
HTTP no es un protocolo seguro porque la transmisión no está cifrada. Las posibilidades de cifrado para transmisiones seguras están dadas por HTTPS (HyperText Transfer Procol Secure). Más acerca de HTTPS aquí.

Traducción de Ruta de acceso

Los servidores traducen el componente de ruta de un URI el recurso del sistema de archivos locales. La ruta URI especificada por el cliente es relativa al directorio raíz del servidor Web (Hosts Virtuales). Por ejemplo el cliente puede solicitar un archivo /mydirectory/myfile a través de una solicitud como:

HTTP """HTTP""" GET ObtenerGET /mydirectory/myfile HTTP/1.1
Host: www.host.com

El servidor asigna el nombre de archivo al directorio raíz del host virtual actual. Si el directorio raíz es /var/www el recurso devuelto será /var/www/mydirectory/myfile de acuerdo a la especificación para su tipo MIME.
